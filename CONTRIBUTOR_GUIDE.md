# SkillMark Contributor Guide

This guide explains how to develop and contribute to SkillMark.

## Development Setup

### Prerequisites
- Node 20+
- Python 3.11+
- Supabase account (free tier)
- Anthropic API key

### Initial Setup

1. **Clone & install dependencies:**
   ```bash
   cd /Users/vic/Personal/skillmark
   npm install --prefix frontend
   cd backend && pip install -r requirements.txt && cd ..
   ```

2. **Configure environment:**
   ```bash
   cp .env.example .env
   # Fill in Supabase and Anthropic keys
   ```

3. **Initialize database:**
   - Run migrations in Supabase SQL editor: `backend/migrations/*.sql`
   - Run seed: `backend/seed/skills.sql`

4. **Start dev servers:**
   ```bash
   # Backend (port 8000)
   cd backend && uvicorn app.main:app --reload

   # Frontend (port 3000, separate terminal)
   cd frontend && npm run dev
   ```

## Code Conventions

### Frontend (React/TypeScript)

- **Strict TypeScript:** No `any`, no suppressed errors
- **Component location:** `src/pages/` for pages, `src/components/` for reusable components
- **API calls:** Use the `api` client from `src/lib/api.ts`
- **State management:** Zustand stores in `src/stores/`
- **Hooks:** Custom hooks in `src/hooks/`, prefixed with `use`
- **Translation:** All user-visible strings use `t('key')` from i18next
- **Path alias:** `@/` maps to `src/`
- **Components:** Use shadcn/ui components from `src/components/ui/`

Example hook:
```typescript
import { useState } from 'react'
import { api } from '@/lib/api'

export function useExample() {
  const [data, setData] = useState(null)
  const [loading, setLoading] = useState(false)

  const fetch = async () => {
    setLoading(true)
    try {
      const res = await api.get('/endpoint')
      setData(res.data)
    } finally {
      setLoading(false)
    }
  }

  return { data, loading, fetch }
}
```

### Backend (Python/FastAPI)

- **Pydantic v2:** Use `model_dump()` not `.dict()`
- **Database:** Access via `get_db()` from `app.db.client`
- **Models:** All schemas in `app/models/`
- **Routers:** API routes in `app/routers/`, grouped by domain
- **Services:** Business logic in `app/services/`
- **Auth:** Use `Depends()` decorators with `require_admin`, `require_manager_or_above`, etc.
- **Errors:** Return appropriate HTTP status codes with `HTTPException`

Example router:
```python
from fastapi import APIRouter, Depends, HTTPException, status
from app.dependencies import require_manager_or_above
from app.models.user import UserOut
from app.db.client import get_db

router = APIRouter(prefix="/example", tags=["example"])

@router.post("", status_code=status.HTTP_201_CREATED)
async def create_item(
    payload: ItemCreate,
    current_user: UserOut = Depends(require_manager_or_above),
):
    db = get_db()
    # your logic here
    return result
```

### Database (PostgreSQL/Supabase)

- **Primary keys:** UUID, generated by Supabase
- **Timestamps:** `created_at` and `updated_at` as TIMESTAMPTZ, auto-managed
- **Foreign keys:** Use explicit hints in PostgREST joins (e.g., `users!allocations_user_id_fkey`)
- **Migrations:** SQL files in `backend/migrations/`, numbered sequentially
- **RLS:** Row-level security policies for multi-tenant data

## Feature Development Workflow

### 1. Create a Branch
```bash
git checkout -b feature/my-feature
```

### 2. Develop & Test Locally
- Make changes to frontend/backend
- Test with dev servers running locally
- Verify with browser at http://localhost:3000

### 3. Frontend Testing Checklist
- [ ] Feature works in browser
- [ ] No TypeScript errors
- [ ] UI looks correct (dark mode + light mode)
- [ ] Mobile responsive (if applicable)
- [ ] Keyboard navigation works
- [ ] Translations added (EN + VI)

### 4. Backend Testing Checklist
- [ ] Endpoint responds correctly
- [ ] Authorization checks work (`require_admin`, etc.)
- [ ] Invalid input is rejected with proper error messages
- [ ] Database queries are efficient
- [ ] Test via `http://localhost:8000/docs`

### 5. Commit & Push
```bash
git add .
git commit -m "feat: describe your feature

- Bullet point 1
- Bullet point 2"
git push origin feature/my-feature
```

### 6. Create Pull Request
Push to GitHub and create a PR with:
- Clear title describing the change
- Detailed description
- Checklist of testing done
- Screenshots for UI changes

## Common Tasks

### Adding a New API Endpoint

1. **Create Pydantic model** in `backend/app/models/`
2. **Add router** or update existing in `backend/app/routers/`
3. **Call from frontend hook** in `src/hooks/`
4. **Create React component** using the hook
5. **Test in dev mode**

### Adding a New Page

1. **Create component** in `src/pages/PageName.tsx`
2. **Add route** in `src/routes/index.tsx`
3. **Add nav item** in navigation component (if needed)
4. **Add translations** in locale files
5. **Test at http://localhost:3000/page-name**

### Updating Database Schema

1. **Create migration** file in `backend/migrations/XXX_description.sql`
2. **Run in Supabase SQL editor**
3. **Verify migrations ran** in Supabase → Database → Migrations
4. **Test backend** sees the new columns/tables

### Fixing a Bug

1. **Reproduce locally**
2. **Identify root cause** (check logs, add console.log/print statements)
3. **Fix in code**
4. **Test thoroughly** — cover edge cases
5. **Commit with clear message:**
   ```bash
   git commit -m "fix: description of the bug and how it's fixed"
   ```

## Debugging

### Frontend
- Use browser DevTools (F12)
- Check Console tab for errors
- Use React DevTools extension
- Add `console.log()` or debugger breakpoints
- Check Network tab for API calls

### Backend
- Check terminal output from `uvicorn`
- Add `print()` or use Python debugger
- Check logs in Supabase Dashboard
- Use FastAPI Swagger UI at `/docs`
- Enable logging: `import logging; logger = logging.getLogger(__name__)`

### Database
- Supabase Dashboard → SQL Editor
- Run queries to inspect data
- Check policies in Database → Policies
- View migrations in Database → Migrations

## Pull Request Guidelines

### Before Submitting
- [ ] Code follows conventions above
- [ ] No console errors or warnings
- [ ] No TypeScript errors
- [ ] Tests pass (if applicable)
- [ ] Translations added
- [ ] Commit messages are clear

### PR Title Format
```
feat: Add team management page
fix: Resolve wishlist state sync issue
docs: Update README with new features
refactor: Simplify matching engine logic
```

### PR Description Template
```markdown
## Description
Brief description of the change.

## Related Issue
Closes #123 (if applicable)

## Testing
- [ ] Feature tested locally
- [ ] Edge cases covered
- [ ] No regressions in other features

## Screenshots
(For UI changes)
```
